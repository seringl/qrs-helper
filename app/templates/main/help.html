Help & How-To

{# ── USER GUIDE ───────────────────────────────────────────────────────── #}

The three creation modes

QR + Short Link (full card) — creates a Bitly short link, a branded QR code pointing at it, and a print-ready card (4.25" × 5.5"). Best choice for flyers, posters, and handouts. Click tracking is included.

QR code only — a branded QR code for a URL you already have. You get the QR exactly as generated (high-resolution PNG + vector SVG) and a full print-ready card with your call-to-action and the destination URL. No short link is created, so clicks are not tracked.

Short link only — just the Bitly short link, with click tracking. Use it for emails, social posts, or anywhere a tidy URL matters.

Creating something, step by step

  1. Open Create from the menu and pick a mode.
  2. Enter a title (for your own reference) and the destination URL.
  3. For short links, choose a branded domain (or the default if allowed).
  4. For QR/card modes, pick a logo and a card template.
  5. Optionally tick Change the call-to-action text to override the template's default.
  6. Click Generate. You'll land on a results page with a preview and downloads.

Print tips

Always test-scan a printed proof before mass printing. Don't shrink the QR code below 0.8" × 0.8" on paper, and keep clear space around it. Dark-on-light scans best — avoid printing the code over photos or busy backgrounds.

{# ── CLICK STATISTICS (REVIEW) ────────────────────────────────────────── #}

Where to find your numbers

Everything click-related lives under the Review menu. Review → Stats is the analytics dashboard; Review → My History lists your items with a click count you can click into for a drill-down. Reporters and admins also get All History and Export CSV.

What the dashboard shows

For the links you select, the Stats page presents:

  • Click trend over time, so you can see when scans happened.
  • Devices — desktop vs. mobile vs. tablet breakdown.
  • Countries and cities where clicks came from.
  • Referrers — where the traffic originated.
  • A "My data" toggle to scope everything to just your own items.
  • CSV export (reporter/admin) for use in a spreadsheet.

Why a new link shows zero clicks

Click data is gathered from Bitly automatically about once an hour, so a brand-new link reads zero until the next harvest. Only short links are tracked — QR-only items have no short link and therefore no click data. An admin can pull fresh numbers immediately from Admin → App Settings → Troubleshooting.

{# ── USER SETTINGS ────────────────────────────────────────────────────── #}

Your settings

Open (your name) → User Settings in the top-right menu.

  • Display name — the name shown for you across the app.
  • Default logo — pre-selected on the Create form so you don't pick it each time (falls back to the global default).
  • Preferred card template — pre-selected on the Create form.
  • Change password — for local accounts. Directory accounts manage their password in the directory.

Custom back-halves

If an admin has granted you permission, the Create form shows a custom back-half field for short links — the bit after the slash (e.g. example.co/SummerConcerts). It needs a branded domain that supports custom links on your Bitly account.

{# ── ADMIN & SETUP ────────────────────────────────────────────────────── #} {# Ordered to follow the path you'd actually take setting the site up. #}

1. First-time setup checklist

After your first login as qrsadmin (and setting a new password), work top to bottom. Each step links to its own section below.

  1. Set branding & global options in App Settings.
  2. Add at least one Bitly domain (or allow the default for testing).
  3. Upload one or more logos and set a global default.
  4. Review the seeded Standard card template, or create your own.
  5. Create the user accounts people will log in with and set their roles.
  6. Create a test card end-to-end and scan it from a phone to confirm it resolves.

2. App Settings & branding

Start here — these global options shape everything else. The right-hand panel holds:

  • Site title / subtitle — shown in the menu bar.
  • Open registration — allow self sign-up (new users get No access until promoted).
  • Allow Bitly default domain — if false, only branded domains may be used.
  • Session expiry days — global auto-logout; 0 = never.
  • File retention days — how long generated files are kept; 0 = forever.
  • Analytics harvest interval (min) — how often clicks are pulled (restart to apply).
  • Analytics retention days — how long click records are kept; 0 = forever.

The Branding card on the left uploads the small logo mark shown next to the site name in the menu bar.

3. Bitly Domains

Add the branded short domains your organisation uses (for example example.co, with no https://). Each domain must already exist on your Bitly account. Mark one as the default, and activate/deactivate as needed. Whether the plain bit.ly domain is allowed is set in App Settings.

4. Logos

Upload the logos users can drop into QR codes (PNG, JPG, or SVG; a PNG/SVG with a transparent background works best). Mark one as the global default, and activate/deactivate logos. The logo is uploaded to QRCode Monkey and embedded into the QR code when one is generated.

5. Card Templates

Templates control how the printed card looks. Each one has a live preview that updates as you change it. Settings:

  • Colours — background, inner panel, and text.
  • Corner style — square for sharp corners, or rounded/custom to use the radius sliders.
  • Outer corner radius — how round the card's outer corners are.
  • Inner corner radius — the white QR panel's corners, set independently of the outer corners.
  • QR code size — how large the QR sits, centred in the panel.
  • CTA text position and URL text position — slide each line up or down to taste.
  • Default CTA text — the call-to-action used unless a creator overrides it.

Every slider has − / + buttons for precise nudging in addition to dragging. Both text lines auto-size to the inner frame width; the CTA uses Inter SemiBold and the URL uses Inter Light.

6. Users & roles

Create the accounts people log in with under Admin → Users. For each local account a temporary password is generated and shown once; the user must change it on first login. From the same screen you set the role, toggle the custom back-half permission, set a per-user session expiry, deactivate accounts, and reset passwords. (You can't demote your own admin account.)

Each account has exactly one role, from least to most access:

  • No access — where self-registered users land until you promote them.
  • User — can create short links, QR codes, and cards, and see their own history and stats.
  • Reporter — everything a user can do, plus All History and CSV export.
  • Admin — full access, including everything in this Admin menu.

Background jobs & Troubleshooting

Two jobs run on their own in the background. The Troubleshooting card on the App Settings page lets an admin run either one immediately:

Run analytics harvest now — contacts Bitly and pulls the latest click counts for every tracked short link, storing them locally so the Stats and History pages are up to date. This normally runs automatically every hour (see the harvest interval setting). Run it manually right after creating a link if you don't want to wait for the next scheduled run.

Run retention cleanup now — deletes generated card/QR files and stored click records that are older than the retention periods in App Settings (File retention days and Analytics retention days), freeing disk space. This normally runs once a day. You'll be asked to confirm because deletion can't be undone.
